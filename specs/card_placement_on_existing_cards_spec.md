# 既存カードクリックによるカード配置仕様

## 概要
手札からカードを選択している状態で、すでにカードが置かれている自分のボード上のスロット（エリア）のカードをクリックした際、そのエリアを選択したものとみなしてカードを配置できるようにします。

## 現状の課題
現在は `Card` コンポーネントで `e.stopPropagation()` が呼ばれているため、カードをクリックしても親要素である `Zone` のクリックイベント（`onZoneClick`）が発生しません。
そのため、カードが置かれているスロットに手札からカードを出したい場合、カード以外の「隙間」をクリックする必要があり、操作性が良くありません。

## 仕様詳細

### 変更対象
`src/board/Board.tsx` の `handleCardClick` 関数

### 新しいロジック

`handleCardClick` 内で以下の条件を満たす場合、カードの選択（`setActiveCard`）ではなく、カードの移動（`moves.moveCard`）を実行します。

1.  **手札のカードを選択中である**
    -   `activeCard` が存在し、かつ `activeCard.location.area` が `AREAS.HAND` であること。
2.  **クリックしたカードが自分のボード上のスロットにある**
    -   クリックされた `location.playerId` が `myID` であること。
    -   クリックされた `location.area` が `AREAS.BOARD` であること。
3.  **移動が有効である**
    -   （既存の `handleZoneClick` と同様のバリデーションが必要ですが、`moves.moveCard` 側でもチェックされるため、ここでは意図の変換を行います）

### 処理フロー

1.  ユーザーが手札のカードAをクリック -> `activeCard` = A
2.  ユーザーが自分のスロットにあるカードBをクリック (`handleCardClick` 発生)
3.  **判定**: `activeCard` は手札か？ -> Yes
4.  **判定**: クリックした場所は自分のボードか？ -> Yes
5.  **アクション**: `setActiveCard(B)` ではなく、`moves.moveCard(A, to: Bの場所)` を実行する。
6.  `activeCard` をクリアする。

### 例外ケース
-   **謀略カード（Guile Tactics）使用中**:
    -   `Traitor` (裏切り) などの処理中は、対象カードを選択する必要があるため、この「配置」ロジックは適用せず、既存のカード選択ロジックを優先します。
    -   `activeGuileTactic` が存在する場合は、既存の挙動を維持します。

## 影響範囲
-   `src/board/Board.tsx`
-   既存のカード選択ロジック（自分の場のカードを確認するためにクリックする場合など）
    -   手札を選択していない状態であれば、これまで通り自分の場のカードを選択（ハイライト）できます。
    -   手札を選択している状態だと、これまでは「選択の切り替え」になっていましたが、今後は「配置」になります。これはユーザーの意図（手札を持っている＝出したい）に合致します。
