# スマホUI対応（レスポンシブ対応）仕様書

## 1. 概要

既存のPC版UIを維持しつつ、スマートフォン（横画面）でも快適にプレイできるようレスポンシブ対応を行う。

## 2. 要件

| 項目 | 内容 |
|------|------|
| **対象画面** | Lobby画面、Game画面の両方 |
| **ターゲット** | スマホ横画面（Landscape） |
| **最小対応幅** | 375px (Mobile M) |
| **操作方法** | PC/スマホ共通（タップ操作、変更なし） |
| **既存PC版** | 維持（同一コードでレスポンシブ対応） |

## 3. レイアウト方針

### 3.1 ブレークポイント

| ブレークポイント | 幅 | 用途 |
|------------------|-----|------|
| `sm` | 640px以下 | スマホ横画面 |
| `md` | 768px以上 | タブレット以上 |
| `lg` | 1024px以上 | PC（既存レイアウト） |

### 3.2 Game画面 (`Board.tsx`)

#### フラグ9列の維持
- フラグ9列は横並びを維持
- 画面幅が足りない場合: **横スクロール** を許容
- `overflow-x-auto` で横スクロール可能にする

#### カードサイズ
- 画面幅に応じてカードサイズを縮小
- 最小サイズ: 横60px × 縦84px 程度（アスペクト比1:1.4維持）
- 横スクロールを許容し、カードの操作性を優先

| 画面幅 | カードサイズ目安 |
|--------|------------------|
| 1024px以上 | 80px × 112px（現状維持） |
| 640px〜1023px | 64px × 90px |
| 375px〜639px | 60px × 84px |

#### ヘッダー/フッター
- 情報量を維持しつつ、コンパクトに再配置
- 不要な余白を削減

### 3.3 Lobby画面 (`Lobby.tsx`)

#### ルーム一覧
- カード型レイアウトを維持
- グリッド列数を画面幅に応じて調整
  - PC: 複数列
  - スマホ: 1〜2列

#### モーダル
- `CreateRoomModal`, `JoinByIdModal`, `JoinRoomModal`
- 最大幅を `max-w-sm` などで制限し、スマホでも見切れないように

## 4. 実装方針

### 4.1 TailwindCSSレスポンシブクラスの活用

```css
/* 例: カードサイズのレスポンシブ対応 */
.card {
  @apply w-12 h-[67px] sm:w-16 sm:h-[90px] lg:w-20 lg:h-28;
}
```

### 4.2 横スクロールコンテナ

```tsx
// 戦場エリアを横スクロール可能に
<div className="overflow-x-auto">
  <div className="grid grid-cols-9 gap-2 min-w-[640px]">
    {/* フラグとゾーン */}
  </div>
</div>
```

### 4.3 変更対象ファイル

| ファイル | 変更内容 |
|----------|----------|
| `src/board/Board.tsx` | 戦場エリアの横スクロール対応、レスポンシブクラス追加 |
| `src/board/Card.tsx` | カードサイズのレスポンシブ対応 |
| `src/board/Zone.tsx` | ゾーンサイズのレスポンシブ対応 |
| `src/board/Hand.tsx` | 手札エリアのレスポンシブ対応 |
| `src/board/Flag.tsx` | フラグサイズのレスポンシブ対応 |
| `src/board/DeckPile.tsx` | デッキ表示のレスポンシブ対応 |
| `src/Lobby.tsx` | グリッド列数調整、モーダル幅調整 |

## 5. 検証項目

- [ ] 375px幅（横画面）でGame画面が操作可能
- [ ] 375px幅（横画面）でLobby画面が操作可能
- [ ] 横スクロールでフラグ9列すべてにアクセス可能
- [ ] カードのタップ操作が正常に動作
- [ ] PC版レイアウトが崩れていない（1024px以上）
- [ ] モーダルが画面内に収まる

## 6. 備考

- スマホ縦画面は今回の対象外（横画面を想定）
- ドラッグ操作やスワイプジェスチャーは導入しない
- 将来的に縦画面対応が必要になった場合は別途検討
