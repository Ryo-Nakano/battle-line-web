# 手札ソート機能 仕様書

## 1. 概要
プレイヤーが自分の手札を整理しやすくするため、手札のカードを所定の規則に従って並び替える機能を追加する。

## 2. ソートロジック

手札内のカードを以下の優先順位で並び替える。

1.  **カードタイプ**:
    *   **部隊カード (Troop)** を先に配置。
    *   **戦術カード (Tactic)** を後に配置。

2.  **部隊カードの並び順**:
    *   **第一キー: 色 (Color)**
        *   `constants.js` の `COLORS` 定義順とする。
        *   順序: `Red` -> `Orange` -> `Yellow` -> `Green` -> `Blue` -> `Purple`
    *   **第二キー: 数値 (Value)**
        *   昇順 (1 -> 10)

3.  **戦術カードの並び順**:
    *   名前 (Name) または ID のアルファベット昇順。

## 3. 実装詳細

### 3.1 サーバーサイド (Game Logic)
*   **Move関数**: `sortHand` を追加する。
    *   引数: なし (現在のプレイヤーの手札を対象とする)
    *   処理: `G.players[ctx.currentPlayer].hand` の配列を上記のロジックでソートする。

### 3.2 フロントエンド (UI)
*   **場所**: 手札エリア (`Hand` コンポーネント) の上部または内部。
*   **UI要素**: 「手札を並び替え」ボタン (アイコンボタン等でも可)。
*   **アクション**: ボタン押下時に `moves.sortHand()` を実行。

## 4. データ構造への影響
*   `G.players[playerID].hand` の配列順序が変更されるのみ。データの追加・削除は発生しない。
