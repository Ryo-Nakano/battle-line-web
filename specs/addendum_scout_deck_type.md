# 追加要望: デッキ戻し（スカウト）時のカードタイプ制限

## 概要
手札からカードをデッキに戻す際、カードの種類と対応するデッキの種類が一致していなければならないように制限する。
（部隊カードは部隊山札へ、戦術カードは戦術山札へのみ戻せる）

## 変更点

### 1. ロジック (`src/moves.js`)
*   `moveCard` のバリデーション変更:
    *   `to.area === 'deck'` の場合:
        *   `card.type` と `to.deckType` を比較する。
        *   不一致の場合は `INVALID_MOVE` を返す（コンソール警告を出力）。

### 2. UI (`src/board/Board.tsx`)
*   **デッキボタンの無効化条件**:
    *   現在: `(!activeCard && Deck.length === 0) || !isMyTurn`
    *   変更後:
        *   基本条件: `!isMyTurn` なら無効。
        *   カード選択時 (`activeCard` あり): `activeCard.card.type` がボタンのタイプと一致しないなら無効。
        *   カード未選択時 (`activeCard` なし): デッキが空なら無効。

## 確認事項
*   部隊カードを選択中、戦術デッキボタンが無効化されていること。
*   戦術カードを選択中、部隊デッキボタンが無効化されていること。
*   正しい組み合わせならボタンが有効で、クリックするとカードが戻ること。
